CONTRIBUTING TO EDULINK

HOW TO CONTRIBUTE



1\. Fork the repository

Click the Fork button on the top-right of the GitHub repository.



2\. Clone your fork

git clone https://github.com/Roniscend/EduLink.git

cd EduLink



3\. Create a new branch

git checkout -b feature/short-description



Examples:

\- feature/add-new-screen

\- fix/navigation-bug



4\. Make your changes

\- Follow the existing project structure

\- Use clean and readable Kotlin code

\- Follow Jetpack Compose best practices

\- Add comments where required



5\. Test your changes

Ensure the app builds and runs correctly.

Verify UI changes and navigation flows.



6\. Commit your changes

git commit -m "feat: short description of changes"



7\. Push to your fork

git push origin feature/short-description



8\. Open a Pull Request

\- Go to the original EduLink repository

\- Click on New Pull Request

\- Clearly explain your changes

\- Attach screenshots if UI is affected

\- Link relevant issues if any



--------------------------------------------------



CODE GUIDELINES

\- Use meaningful variable and function names

\- Keep composables modular and reusable

\- Avoid unused imports and resources

\- Follow MVVM architecture where applicable
